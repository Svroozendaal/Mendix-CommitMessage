# PHASE_2_PLANNING
## Goal

Create an actionable technical plan with explicit contracts and risks.

## Entry Criteria

1. `development/AGENTS.md` has been read.
2. Clarifying workflow questions have been asked as needed.

## Required First Action

1. Read `development/AGENTS.md`.
2. Ask workflow questions first.
3. Ask which skills should be used and suggest relevant defaults.

## Tasks

1. Define scope and non-goals.
2. Produce file-level change plan.
3. Document assumptions and risks in `DECISIONS_LOG.md`.
4. Hand off to Implementer with clear acceptance criteria.
5. Pause at `WAIT_FOR_APPROVAL` before any implementation changes, unless `AUTO_APPROVE` is explicit.

## Exit Criteria

1. Scope, non-goals, and acceptance criteria are documented.
2. File-level plan is ready.
3. Approval checkpoint status is clear.
